## 블로킹 IO 테스트 

- 싱글 스레드 모델 기반 블로킹 IO 
- 입력한 값을 대문자로 변환후 output 

요청 예시)

```text
~  nc 127.0.0.1 8080
hi
HI
test
TEST
tttt
TTTT
```

특징 

- Blocking 소켓 특징에 따라 IO 처리시 스레드 블로킹이 발생하며 때문에 동시에 여러 클라이언트에 대한 처리 불가능 
- Blocking 되는 메서드 
  - `accept()`
  - `read()`
  - `write()`
- 호출한 스레드는 소켓 연결이 열리거나 데이터 입출력이 완료될 때까지 해당 스레드를 Blocking 
- 싱글 스레드 모델 기반이며 당연히 다중 클라이언트 요청 처리 불가능
